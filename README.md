# kongyo2/TDD

Claude Codeで厳格な**テスト駆動開発**（以下、TDD：Test-Driven Development）を実践するための専用プラグインです。
このプラグインを導入すると、Claudeは指示された変更に対してテストを自動的に先行作成し、段階的に実装を進めるようになります。 
主にTypeScriptと**Vitest**を採用したプロジェクトに向けて最適化されています。

---

## 特徴

本プラグインは、エージェントが自律的に正しい開発サイクルを維持できるよう以下の仕組みを備えています。

- **テストファーストの強制**：テストが失敗する前にプロダクションコード（本体の実装コード）を書く行為を禁止します。
- **Vitest実行の安全化**：ウォッチモードによるプロセスフリーズを防ぐため、`vitest run`の明示的な実行を強制します。
- **仮実装と三角測量の重視**：最初に定数を返す最小限の実装（仮実装）を行い、複数のテストケースによって処理を一般化（三角測量）するアプローチを徹底させます。
- **テスト用アンチパターンの排除**：テストを書かずに実装を済ませる後追いテストや、内部実装に依存した壊れやすいテストコードの発生を防ぎます。

---

## 提供されるコンポーネント

本プラグインを有効にすると、以下の構成要素がClaudeの環境に登録されます。

- **`tdd` スキル**：TDDプロセスの基本方針やVitestの実行基準をまとめたスキルです。
  - **`test-style`**：Arrange、Act、Assertによる記述法や、わかりやすいテスト命名規則について解説します。
  - **`mocking`**：外部APIやファイルシステムなどの境界（バウンダリ）のみをモックにする規律を定めます。
  - **`vitest`**：エージェント環境下で安全にVitestを起動するためのCLIオプションやAPIの使い方を網羅します。
  - **`anti-patterns`**：避けるべきテストの書き方と、それを正すための具体的な修正方法を示します。

## 開発プロセス（エージェントが辿る5つのステップ）

本スキルを実行したClaudeは、以下の手順を繰り返して進捗を報告します。

1. **Explore（探索）**：実装に先立ち、既存のプロダクションコードとテストコードを把握します。
2. **Plan（計画）**：追加・修正すべき仕様を最小単位の挙動に分解し、テストファイルに `it.todo` でプレースホルダーを作成します。
3. **Red（テストの失敗確認）**：テストを1つだけ実装し、意図した原因でアサーションが失敗（Red）することを確認します。
4. **Green（最小限の実装）**：テストをパスさせるために必要最小限の実装コードを書きます。
5. **Refactor（リファクタリング）**：重複を排除し、コードの設計を整えます。変更を加えるたびにテストを実行して動作を保証します。
